# N3 universe + deep-history backfill — coverage audit

**Date:** 2026-06-12 · **SoT item:** N3 · **Branch:** `feat/n3-universe-backfill`

## Goal

Acceptance artifact for the N3 deep-history backfill: prove that the committed
25-perp research universe (`config/universe.toml`) is ingested into
`analytics.db` with full OHLCV depth across 1h/4h/1d/**1w** (plus 15m for the
three majors), matching funding-rate history, and survivorship-aware
`symbol_lifecycle` metadata — with no gaps.

## What ran

- `make universe-backfill` — `analytics backfill --universe --timeframes 1h 4h
  1d 1w --since 2019-01-01` over all 25 symbols. Binance offline path
  (`DATA_SOURCE` unset → binance), upsert-only (never deletes), no Telegram.
  Completed clean, 25/25 symbols, zero failures.
- `analytics backfill --symbols BTCUSDT ETHUSDT SOLUSDT --timeframes 15m
  --since 2019-01-01` — majors 15m deepened to listing.
- Each run also refreshed `symbol_lifecycle` from futures exchangeInfo (all 25
  resolve `TRADING`).

## Result (acceptance ✓)

- **25 symbols**, **1,945,442 OHLCV rows**, timeframes `15m / 1h / 4h / 1d / 1w`.
- **`gap_pct = 0.0` for every (symbol × timeframe) cell** — no missing bars
  against the expected count between each cell's first and last bar.
- Funding-rate history present for all 25 from listing (137,057 rows total).
- `symbol_lifecycle`: 25 rows, all `TRADING`, onboard dates populated.

## Sanity checks

- **Majors deepened to listing** — BTC 15m/1h start 2019-09-09, ETH 2019-11-27,
  SOL 2020-09-14, matching their USDT-M perp onboard dates.
- **Younger symbols start at their onboard date, not 2019** — HYPE 2025-05-30,
  VVV 2025-01-29, PAXG 2025-03-27, ENA 2024-04-02, TAO 2024-04-11, TON
  2024-03-01, ONDO 2024-01-20. These short histories are correct (the symbol
  did not exist earlier), **not** coverage gaps.
- **1w anchoring** — weekly `first_day` is the Monday on/before each listing
  (Binance weekly opens Mon 00:00 UTC), e.g. BTC `2019-09-02`, so the weekly
  bar count is one or two ahead of a naive listing-date division. Expected.
- **Zero gaps is genuine**, not a math artifact: `expected` is computed as
  `(last_ms − first_ms) // tf_ms + 1` per cell, and Binance USDT-M perps have
  effectively continuous coverage over this window.

## Survivorship caveat

The `symbol_lifecycle` guard is **forward-looking from the 2026-06-12 snapshot**.
All 25 universe members are current survivors (`TRADING`); there are no
`DELISTED` rows yet. Perps that were delisted *before* this snapshot are not
enumerable from the free Binance API, so they cannot be back-filled into the
lifecycle table. Going forward, any tracked symbol that disappears from
exchangeInfo is marked `DELISTED` with a sticky `delisted_noted_ms` and its
OHLCV/funding rows are **never deleted** (noted, not dropped) — so the research
breadth set becomes progressively survivorship-correct as the universe rotates.
